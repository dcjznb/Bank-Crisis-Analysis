# Bank-Crisis-Analysis

Python Project 2 for AF3214

## 项目简介

这是一个自动化事件研究系统，用于分析硅谷银行（SVB）倒闭引发的系统性传染效应。

本项目使用 Python 构建量化金融中的经典模型: 事件研究法（Event Study）。

核心研究问题是: 2023 年 3 月 SVB 的突然倒闭，是否在市场中引发了非理性恐慌，并对基本面相对正常的区域性银行造成了传染性冲击。

本工具完成了以下工作:

1. 抓取并清洗真实市场交易数据。
2. 用统计模型剥离大盘波动干扰。
3. 提炼由 SVB 事件导致的额外市值蒸发（恐慌溢价）。

## 核心方法与数学原理

本项目采用基于市场模型（Market Model）的事件研究法。

### 1. 正常预期收益（Expected Return）

假设每只银行股在平时与整体大盘（标普 500）存在稳定联动关系，通过线性回归得到:

$$
E(R_{i,t}) = \alpha_i + \beta_i R_{m,t}
$$

其中:

- $\alpha_i$: 常数项。
- $\beta_i$: 个股相对大盘的波动敏感度。

### 2. 异常收益率（Abnormal Return, AR）

在 SVB 倒闭期间，用股票真实收益减去模型预期收益，差值即事件冲击:

$$
AR_{i,t} = R_{i,t} - E(R_{i,t})
$$

### 3. 累计异常收益率（Cumulative Abnormal Return, CAR）

将事件窗口内每日 AR 累加，观察恐慌在市场中的蔓延与累积过程。

## 代码执行逻辑（Step-by-Step）

为保证报告结构清晰，代码分为四个功能模块。

### 第一部分: 数据获取与清洗（Data Collection）

做什么:

- 设定研究对象: WAL、ZION、CMA。
- 设定市场基准: 标普 500 指数。

分析思路:

- 不直接比较绝对股价。
- 将价格序列统一转换为日收益率，保证横向可比性。

### 第二部分: 时间窗口与模型计算（Event Study Calculation）

做什么:

- 划分和平时期与危机时期。
- 估计模型参数并计算恐慌冲击。

分析思路:

- 估计窗口（Estimation Window）: Day 0（3 月 9 日）前 120 天到前 20 天。
- 在估计窗口中拟合每只银行的 $\alpha$ 和 $\beta$。
- 事件窗口（Event Window）: Day -5 到 Day +15。
- 将参数代入事件窗口，计算每日 AR 并累加为 CAR。
- 若 Day 0 前 CAR 已明显为负，支持信息泄漏（Information Leakage）假设。

### 第三部分: 数据可视化（Data Visualization）

做什么:

- 将 CAR 结果转化为直观折线图。

分析思路:

- 黑色垂直虚线表示 SVB 危机爆发日（Day 0）。
- 若 Day 0 后多只银行 CAR 同步显著下行，说明恐慌效应具备系统性。
- 例如 WAL 在样本中出现接近 80% 的累计额外跌幅，显示冲击远超大盘常规波动。

### 第四部分: 统计显著性检验（Hypothesis Testing）

做什么:

- 使用 T 检验（T-Test）验证结论是否具有统计显著性。

分析思路:

- 计算 Day +5 的平均累计异常收益率（CAAR）。
- 得到对应 P-Value（P 值），检验是否可由随机波动解释。

结论应用:

- 当 $p < 0.05$ 时，可在 5% 显著性水平上拒绝“纯随机波动”假设。
- 本项目结果显示 SVB 危机引发的系统性传染效应真实存在，而非市场噪音。
